What AI tools are typically used for your role?
AI tools such as AI-integrated editors, GitHub Copilot, agentic CLI tools or chatbots including OpenAI, Claude and Gemini.

What are the benefits and risks of using AI in a professional setting?
Benefits:
- Can get prototypes very quickly for example database schemes.
- Can instantly generate unit and integration testing
- Can generate documentation or code comments using code logic very quickly
Drawbacks:
- There will be major security vulnerabilities; for example AI could suggest code that could succumb to SQL injection.
- AI can easily hallucinate and give code that does not do the job properly or is very inefficient.
- Overly relying on AI can lead to code that works but is not maintainable going forward.

What types of information should never be entered into AI tools?
- Proprietary source code should never be input into AI tools.
- The entire company repo should not be pasted into AI tools.
- API keys should never be input into AI.
- User data should not be entered into AI.
- Company strategies should not be input into AI.

How can you fact-check and validate AI-generated content to ensure accuracy?
Once I receive the AI generated code, I can explain it back to myself to see if I fully understand the code, if I do not I should use the code. If I am using AI, I should test the scripts in a local, isolated environment before integrating AI generated scripts into the main branch.

By keeping and adhering to ethical rules. I will keep a rule where I attemp to solve a logic or fix a bug by myself for set amount of time before considering asking AI. I will use AI to speed up my work and not create it from start.
